## Arduino UNO 引脚分配

> ✅ 已按同款小车的实测项目 [lxbme/arduino-balance-car](https://github.com/lxbme/arduino-balance-car)
> 修正（套件接线固定，不可自选）。

| UNO 引脚 | 连接 | 说明 |
|---|---|---|
| D2 (INT0) | 左编码器 A 相 | 外部中断，CHANGE 触发（2 倍频） |
| D5 | 左编码器方向（B 相） | 中断内读电平判向 |
| D4 | 右编码器 A 相 | ⚠️ 无外部中断，需 **PCINT**（PinChangeInt 库）——朋友项目禁用了右编码器，我们必须启用才有可用里程计 |
| D8 | 右编码器方向（B 相） | 中断内读电平判向 |
| D3 | 按键 KEY | 启停切换（因此 D3 不能用作编码器中断） |
| D12 / D13 | TB6612 IN1 / IN2 | 左电机方向 |
| D7 / D6 | TB6612 IN3 / IN4 | 右电机方向 |
| D10 (PWM) | TB6612 PWMA | 左电机 PWM |
| D9 (PWM) | TB6612 PWMB | 右电机 PWM |
| A4 (SDA) / A5 (SCL) | MPU6050 | I2C（地址 0x68） |
| A0 | 电池分压采样 | 电压 = ADC 均值 × **0.05371**（朋友项目实测系数） |
| D0/D1 (RX/TX) | RDK X5 UART | ⚠️ 与 USB 烧录复用，烧录时断开；UNO TX→X5 RX 需分压到 3.3V |

STBY 在套件板上已处理（无独立引脚），无需软件控制。

## 与朋友项目的差异（重要）

1. **右编码器必须启用**：朋友的版本单编码器就能平衡，但导航需要双轮里程计。
   方案：D4 用 PCINT 中断（PinChangeInt 库），A 相 CHANGE 触发 = 2 倍频
2. **通信改 UART**：朋友走「UNO →I2C→ ESP32 →micro-ROS」，我们按任务书要求
   走「UNO →UART(D0/D1)→ RDK X5」；调试输出与协议共用一个串口，
   固件里调试打印须在联调时关闭（波特率统一 115200，朋友的 9600 不沿用）
3. **雷达直连 RDK X5**：LD08 UART 直接接 X5（或 USB 转串口），不经 ESP32 中转

## 接线注意事项

1. **共地**：UNO、TB6612、RDK X5、ESP32-S3 所有 GND 必须连通
2. **电平**：UNO 是 5V 系统，RDK X5 / ESP32-S3 是 3.3V —— UNO TX 出去的信号要分压或电平转换
3. **舵机独立供电**：舵机 5V 单独一路 DC-DC，只共地，防止堵转电流拉崩逻辑电源
4. **RDK X5 供电**：峰值电流大，DC-DC 至少 5V/3A，线径要够
5. **LD08 供电**：5V，注意电机启动电流
6. **MPU6050**：套件板载位置固定；机械中值参考朋友项目 TARGET_ANGLE=-2.3°（本车需重测）
